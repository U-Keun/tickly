# Tickly 버그 트리아지 (최소)

## 입력(없으면 요청)
- 증상(1~2줄)
- 재현(1~3단계)
- 기대/실제(각 1줄)
- 로그(30~80줄 이내)
- 관련 코드: 1~3개 함수/컴포넌트(가능하면)

## 출력(반드시 아래 순서)
A) Hypotheses (정확히 3개, 각 1줄)
B) Experiments (정확히 3개, 각 1줄)
C) Most likely cause (1줄)
D) Minimal fix (diff, 변경 40줄 내 권장)
E) Questions (0~2개)

## 규칙
- 레포 전체 요청 금지.
- 설명은 짧게, 실험/결정/수정에 집중.
